# 📦 تعديلات آخر جولة — متابعة الصفقات على مدار الساعة (إصلاح عدم وصول تحديث TP1)

> ⚠️ تُطبَّق **فوق** الحزم السابقة. تعتمد على ملفات الجولات السابقة.

## 🔍 لماذا لم يصلك تحديث الصفقة رغم ضرب TP1؟ (التشخيص الكامل)
فحصت المستودع الحيّ على GitHub. تعديلاتي موجودة فعلاً هناك، لكن المشكلة كانت في
**الجدولة + فلتر الجلسة**، سببان متراكبان:

### 1) فجوة 3 ساعات في الجدولة (cron)
كان cron تحديث الصفقات:
```
0 0-19 * * 1-5   → 00:00–19:59 UTC فقط
0 23 * * ...     → الساعة 23 UTC
```
فالفترة **20:00–22:59 UTC (= 23:00–01:59 محلي) بلا أي تشغيل**. صفقتك ضربت TP1 في هذه الفجوة.

### 2) فلتر الجلسة يوقف المتابعة بعد 22:00 محلي
حتى داخل ساعات العمل، كان `update_outside_trading_hours = false`، فالسكربت يخرج
فور انتهاء جلسة الإشارات (22:00 محلي) دون متابعة الصفقات المفتوحة.

> الذهب يُتداول 24/5، وصفقة مفتوحة قد تضرب TP1/SL/تريلينغ في أي ساعة.

## ✅ الإصلاح
1. **`update_trades.yml`**: التشغيل الآن **كل 30 دقيقة على مدار الساعة** (الإثنين–الجمعة + افتتاح الأحد).
   - توليد **الإشارات الجديدة** يبقى مقيّداً بالنافذة 02:00–22:00 (لم يتغيّر — `analyze.yml`).
   - **إدارة الصفقات القائمة** فقط هي التي صارت 24/5.
2. **`config.json`**: `trade_management.update_outside_trading_hours = true` → المتابعة لا تتوقف خارج النافذة.
3. **`run_trade_updates.py`**: ضبط رسالة الحالة (heartbeat) لتُرسَل **مرة كل ساعة تقريباً** (وليس مرتين)
   بما أن التشغيل صار كل 30 دقيقة. أحداث TP1/SL/تريلينغ الفعلية تُرسَل **فوراً دائماً**.

## 📊 الأثر
- صفقة تضرب TP1/SL/تريلينغ في أي وقت → تصلك الرسالة خلال ≤30 دقيقة.
- رسالة حالة مختصرة كل ساعة لكل صفقة مفتوحة.
- لا فجوات ليلية في المتابعة.

## ⚙️ تحكّم
- `notifications.heartbeat_interval_minutes` (افتراضي 60): الفاصل بين رسائل الحالة.
- `notifications.heartbeat_on_trade_update = false`: إيقاف رسائل الحالة (الأحداث تبقى).
- لإرجاع المتابعة للنافذة فقط: `update_outside_trading_hours = false`.

## 📂 الملفات (3)
- `.github/workflows/update_trades.yml` — جدولة 24/5.
- `config.json` — `update_outside_trading_hours: true` + `heartbeat_interval_minutes`.
- `scripts/run_trade_updates.py` — ضبط تكرار الـ heartbeat.

## 🚀 الرفع المهم

   صفقتك التي ضربت TP1 فوراً ويرسل التحديث (لأن المتابعة صارت تعمل خارج الجلسة).
3. تحقّق: `python -m pytest tests/ -q` → المتوقع **308 passed**.
