# 📦 تعديلات آخر جولة — رسالة حالة الصفقات الساعية (مختصرة)

> ⚠️ تُطبَّق **فوق** الحزم السابقة. تعتمد على ملفات الجولات السابقة.

## 🎯 المشكلة
طلبت سابقاً تحديثاً للصفقات المفتوحة **كل ساعة على الأقل حتى لو لم يصل للهدف**،
لكنك لا ترى شيئاً. السبب: الرسالة الساعية كانت مشروطة بـ
`notify_on_trade_update` (الافتراضي = `false`) → فلا تُرسَل على التشغيل المجدول إطلاقاً.

## ✅ الحل
- مفتاح جديد **`heartbeat_on_trade_update` مفعّل افتراضياً** → كل تشغيل ساعي يرسل
  **رسالة حالة مختصرة** حتى بدون أي حدث TP/SL.
- تُكتم تلقائياً في وضع نهاية اليوم (`EOD_QUIET`) لتجنّب التكرار مع التقرير اليومي.

## 📊 الشكل المختصر (كما طلبت + إضافات مفيدة)
```
🔄 Trades Update · 14:00 UTC
Price 4136.12 · 2 open
───────────────
🔴 SELL #7cf3f415  -113 pts (-11.3$) · 0%➜TP1
🟢 BUY  #a1b2c3d4  +48 pts (+4.8$) · 36%➜TP1 [TP1_HIT]
───────────────
🔴 Net: -65 pts (-6.5$)
```
لكل صفقة سطر واحد: **الاتجاه + رقم مختصر + الربح/الخسارة (نقاط ودولار) + التقدّم نحو TP1 + الحالة**،
وسطر **الإجمالي (Net)**. وعند عدم وجود صفقات: `📭 No open trades.`

## 📂 الملفات (3)
- `scripts/run_trade_updates.py` — منطق النبض الساعي + دالة `_build_status_message`.
- `config.json` — `notifications.heartbeat_on_trade_update = true`.
- `tests/test_hourly_status.py` — 4 اختبارات جديدة.

## ⚙️ تحكّم
- لإيقاف النبض الساعي تماماً: اجعل `heartbeat_on_trade_update = false` في `config.json`.
- أحداث TP/SL الفعلية ما زالت تُرسَل كرسائل منفصلة فور وقوعها (لم تتغيّر).
