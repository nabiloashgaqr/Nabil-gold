# 📦 تعديلات آخر جولة — نقل الستوب المبكر (+100 نقطة) + Trailing قبل TP1

> ⚠️ تُطبَّق **فوق** الحزم السابقة. تعتمد على ملفات الجولات السابقة.

## 🔍 ماذا كان يحدث قبل التعديل؟ (إجابة سؤالك)
نقل الستوب للدخول والـ trailing كانا **ينتظران ضرب الهدف الأول TP1**، وليس بعد 100 نقطة.
وبما أن TP1 يقع عملياً حوالي **267 نقطة** (R:R 1.33 مع وقف 200 نقطة)، فالحماية
المبكرة لم تكن تعمل أبداً قبل ربح كبير.

## ✅ بعد التعديل (كما طلبت)
1. **عند +100 نقطة ربح** → يُنقل الستوب إلى الدخول (breakeven) **فوراً دون انتظار TP1**.
2. **بعد النقل** → يبدأ trailing بمسافة **100 نقطة** خلف السعر، يتحرك فقط بالاتجاه الرابح ولا يتراجع.
3. **الإغلاق الجزئي 50% عند TP1 يبقى كما هو**.
4. عند ارتداد السعر للستوب المتحرك → إغلاق كـ **WIN** بالربح المقفول.

### مثال (شراء من 4000)
| السعر | الربح | ما يحدث |
|---|---|---|
| 4005 | +50 | لا شيء بعد |
| 4010 | +100 | 🔒 نقل الستوب للدخول 4000 |
| 4015 | +150 | 📈 trailing → الستوب 4005 |
| 4020 | +200 | 📈 trailing → الستوب 4010 |
| ارتداد لـ4010 | — | ✅ إغلاق WIN بربح +100 مقفول |

## ⚙️ الإعدادات (config.json → trailing_stop)
```jsonc
"trailing_stop": {
  "enabled": true,
  "early_breakeven_points": 100,   // نقل الستوب للدخول عند هذا الربح (0 = تعطيل)
  "trailing_distance": 100,        // مسافة التريلينغ خلف السعر
  "trailing_step": 5,              // أقل تحرك قبل تحديث الستوب (تجنّب تحديثات متكررة)
  "min_profit_lock": 0
}
```
كله بالنقاط (10 نقاط = 1$ على الذهب). غيّر القيم وقت ما تشاء دون تعديل برمجي.

## 📂 الملفات (3)
- `agents/open_trades_manager.py` — منطق النقل المبكر + trailing قبل TP1.
- `config.json` — `early_breakeven_points: 100` و `trailing_distance: 100`.
- `tests/test_early_breakeven.py` — 8 اختبارات جديدة.

## 💡 اقتراحات إضافية (اختيارية — أخبرني إن أردتها)
- **min_profit_lock**: قفل حد أدنى من الربح فوق الدخول (مثلاً 20 نقطة) بدل الدخول الصافي.
- **تريلينغ مرحلي**: مسافة أوسع في البداية (150) ثم أضيق بعد +300 نقطة (لقفل أرباح أكبر).
- **نقل ثانٍ**: عند +200 نقطة انقل الستوب إلى +100 (قفل نصف الربح) قبل بدء التريلينغ.
