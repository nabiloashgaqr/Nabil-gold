## 🔬 السبب الجذري (مفاجئ)
ليس بسبب أخبار ForexFactory الحيّة (الـstub في conftest يعمل ويرجع 0 أحداث).
بل بسبب منطق **`NewsRiskAgent._session_risk()`**: جلسة **"Late NY / Rollover"**
(21:00–23:59 UTC، فترة تدوير منخفضة السيولة) ترفع الحالة إلى **CAUTION** بصرف
النظر عن الأخبار. فالاختبار كان **ينجح نهاراً ويفشل قرب وقت الرولأوفر** — أي
حسب ساعة تشغيلك للـActions.

> هذا سلوك **صحيح** في الكود (تحذير من فترة منخفضة السيولة)، لكن الاختبار كان
> يفترض خطأً أن النتيجة دائماً SAFE بدون أخبار.

## ✅ الإصلاح
تثبيت وقت التشغيل في الاختبار عند **10:00 UTC (جلسة لندن الهادئة)** بدل الوقت
الحالي، فيصبح الاختبار **حتمياً** ويفحص منطق الأخبار فقط (هدفه المعلن):
```python
safe_now = datetime(2026, 6, 23, 10, 0, tzinfo=timezone.utc)
result = agent.check(now=safe_now)
```
طُبّق نفس التثبيت على اختبار DANGER لمتانة أعلى.
